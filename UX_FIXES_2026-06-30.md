## Deferred — cross-repo live launchd wrappers (NOT in this repo; `~/dev/admin` is dirty)

These are live scheduled-job scripts on the machine; `~/dev/admin` has uncommitted
WIP on `finance-surface-daily.sh`. Patches below are ready to apply by hand —
not auto-committed, to avoid clobbering that WIP and changing live automation
unreviewed.

### routines-die-on-usage-limit (HIGH)
`finance-surface-daily.sh` gates success on the `claude -p` exit code only, but a
weekly-usage-limit refusal exits 0 -> the wrapper writes a healthy heartbeat for a
day where nothing ran. Fix: capture stdout, and if it matches the usage-limit
refusal (or lacks the expected status line), force `status=failed`, skip the
healthy heartbeat, and for capture do NOT advance `.finance/last-capture.json` so
the window re-scans. Same gap in the (missing) `finance-capture-daily` wrapper.

### capture-routine-cannot-scan (HIGH)
The daily fact-capture routine is told to grep the transcript, but headless Bash is
denied and the Read tool refuses files over 256KB/25k tokens while transcripts run
4-9MB, so it silently degrades to a start/middle/end sample. Fix: have the wrapper
pre-slice the target transcript to the capture window and strip tool-schema/telemetry
lines before passing the path (or allowlist a read-only grep/jq for that routine),
and make a complete scan a hard requirement that fails loudly instead of sampling.
